⏩ Forward Pass — Complete Reference

Kya kya hota hai · Kaunse tools · Kab use · Special cases
1. Tera Sawaal — Direct Answer
✅ Haan — Input → Numbers ke baad HAMESHA Forward Pass hota hai.
Koi exception nahi. Chahe image ho, text ho, numbers ho — convert ke baad seedha forward pass.

Forward Pass ek single step nahi hai — yeh ek container hai jiske andar multiple operations hote hain. Har operation ek "tool" hai.
Forward Pass = Container
Jaise "cooking" ek container hai jiske andar: chop karna, boil karna, fry karna, season karna hota hai — alag alag tools.

Forward Pass ke andar: Matrix Multiply, Activation, Normalization, Attention, Dropout, Softmax — yeh sab tools hain. Kaunse tools use honge — task pe depend karta hai.
2. Forward Pass — Hamesha Hone Wale Steps
Pre-Step — Input → Numbers Before Forward Pass
Yeh forward pass ka hissa nahi — prerequisite hai. Bina iske forward pass possible nahi.
Image: pixels → tensor (H×W×C)
Text: words → IDs → embedding vectors
Numbers: normalize → vector [0-1 range]
↓ Forward Pass Shuru
Step A — Linear Transformation (Har Layer Mein) Always · Every Layer
Yeh sabse core operation hai. Koi bhi network ho — yeh hamesha hoga. Har layer mein ek baar.
Formula: Z = X · W + b
X = input (numbers ki matrix)
W = weight matrix (trained values)
b = bias vector
· = matrix multiplication — inner dims match karni chahiye
↓
Step B — Activation Function (Har Hidden Layer Mein) Always · Hidden Layers
Linear transformation ke baad non-linearity add karo. Bina activation — sab layers ek hi layer ke barabar hain. Yeh AI ko complex patterns seekhne deta hai.
Common options:
ReLU: max(0, z) — negative → 0, positive → same. Most common.
Sigmoid: 1/(1+e⁻ᶻ) — output 0 to 1. Binary classification.
Tanh: (eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ) — output -1 to 1. RNNs mein.
GELU: Smooth ReLU. Transformers mein.
↓
Step C — Output Layer (Last Layer Only) Always · Last Layer
Last layer mein activation alag hoti hai — task pe depend karti hai.
Classification: Softmax → probabilities (sum=1)
Binary: Sigmoid → 0 or 1 probability
Regression: None → raw number (price prediction)
Multi-label: Sigmoid per class → multiple 1s possible
3. Tools — Hamesha Use Hone Wale
Tool Formula Kya Karta Hai Kab Practice
Matrix Multiply X · W Input ko transform karo. Har neuron ka output nikalo ek saath. Har layer ✓ Practice
Bias Add Z + b Offset add karo. Neuron zero input par bhi fire kar sake. Har layer ✓ Practice
ReLU max(0,z) Negative values zero kar do. Non-linearity add karo. Hidden layers ✓ Practice
Softmax e^z/Σe^z Raw scores → probabilities. Sum = 1 hamesha. Last layer (classification) ✓ Practice
Dot Product Σ(aᵢ×bᵢ) Do vectors ka similarity score. Har neuron andar se yahi karta hai. Every multiply ✓ Practice
4. Special Case Tools — Task Pe Depend

Dropout p(keep)=0.8 Random neurons off karo. Overfitting rokta hai. Training mein only Inference mein BYPASS — sab neurons on

📌 Key Insight — Yaad Rakho
Forward Pass = Container.
MatMul + Activation = Core (hamesha).
Baki tools = plugins (task pe lagao).

Cosine Similarity ek plugin hai — image classification mein nahi lagta, text similarity mein lagta hai.
Dropout ek plugin hai — training mein lagta hai, inference mein nahi.
Attention ek plugin hai — transformer mein lagta hai, simple MLP mein nahi.
